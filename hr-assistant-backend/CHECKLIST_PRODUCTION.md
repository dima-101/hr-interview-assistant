# CHECKLIST_PRODUCTION.md

## 1. Архитектура

- [ ] Уточнён тип хостинга у провайдера.
- [ ] Понятно, можно ли запускать Python/FastAPI на этом хостинге.
- [ ] Принято решение: единый FastAPI-сайт или frontend + отдельный backend API.
- [ ] Выбран поддомен 3-го уровня.

## 2. Домен и сеть

- [ ] Создан поддомен.
- [ ] Настроена DNS-запись.
- [ ] Подключён HTTPS/SSL.
- [ ] Сервис открывается по внешнему адресу предприятия.

## 3. Backend

- [ ] `uvicorn` запускается без ошибок.
- [ ] Все основные маршруты открываются:
  - [ ] `/`
  - [ ] `/about`
  - [ ] `/vacancies`
  - [ ] `/interview?vacancyid=sales_soft_client_001`
  - [ ] `/report`
  - [ ] `/hr/login`
  - [ ] `/hr`
  - [ ] `/reports`
- [ ] Логи не содержат критических ошибок.
- [ ] `.env` вынесен из кода.
- [ ] Секреты не хранятся в git.

## 4. Frontend

- [ ] Интерфейс соответствует корпоративным цветам.
- [ ] На сайте есть ссылка на основной сайт предприятия.
- [ ] С основного сайта предприятия есть ссылка на сервис.
- [ ] Навигация работает.
- [ ] Кнопки и формы доступны и понятны пользователю.
- [ ] Страница отчёта открывается корректно.
- [ ] На мобильном экране интерфейс не ломается.

## 5. Контент

- [ ] Названия вакансий отображаются на русском.
- [ ] Vacancy ID отображаются корректно.
- [ ] Блоки вопросов подгружаются корректно.
- [ ] Отчёт содержит понятные поля.
- [ ] Архив отчётов открывается.
- [ ] Сохранённые `.md`-отчёты читаются без ошибок.

## 6. Надёжность

- [ ] При перезапуске сервера сервис поднимается автоматически.
- [ ] Настроен `systemd`, `supervisor` или другой менеджер процессов.
- [ ] Nginx корректно проксирует приложение.
- [ ] Есть резервная копия проекта.
- [ ] Есть инструкция для сотрудника предприятия.

## 7. Безопасность

- [ ] Отключён debug-режим.
- [ ] Используется HTTPS.
- [ ] Секретные ключи лежат в `.env`.
- [ ] Настроены права доступа на сервере.
- [ ] Внешние API вызываются только по HTTPS.
- [ ] Доступ к HR‑разделу защищён авторизацией.

## 8. Что можно сделать позже

- [ ] Реальное сохранение ответов кандидата.
- [ ] Настоящий backend API интервью.
- [ ] Генерация отчёта не из заглушки, а из реальных ответов.
- [ ] Хранение истории интервью.
- [ ] Экспорт отчёта в PDF/DOCX.
- [ ] Более гибкая система HR‑ролей и пользователей.
