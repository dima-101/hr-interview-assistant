# Project Notes for Submission

## Название проекта
**HR Interview Assistant**

## Краткое описание проекта

HR Interview Assistant — это учебный AI‑проект для проведения структурированных интервью с кандидатами по разным вакансиям.

Проект построен как модульная система, в которой:
- общая логика работы ассистента вынесена в отдельный системный промпт;
- общая информация о компании и принципах оценки кандидатов вынесена в отдельную базу знаний;
- каждая вакансия описана в отдельной карточке профиля;
- итоговая оценка кандидата оформляется по единому шаблону.

Такой подход делает проект удобным для масштабирования: новые вакансии можно добавлять в систему как отдельные карточки без переписывания всей архитектуры.

## Цель проекта

Цель проекта — показать, как AI‑ассистент может помогать HR‑специалисту или интервьюеру:
- выбирать нужный профиль вакансии;
- проводить интервью по заранее заданной структуре;
- задавать релевантные вопросы;
- фиксировать сильные стороны и риски;
- формировать единый итоговый отчёт по кандидату.

## Что реализовано

В проекте подготовлены следующие материалы:

### 1. Системный промпт
Файл: `system_prompt_core.md`

Содержит:
- роль ассистента;
- логику выбора вакансии;
- правила работы с несколькими карточками;
- динамическую нумерацию профилей вакансий;
- структуру интервью;
- шаблон логики формирования итогового отчёта.

### 2. Общая база знаний компании
Файл: `company_knowledge_base.md`

Содержит:
- описание типа компании и её направлений;
- типы вакансий;
- общие принципы оценки кандидатов;
- подход к софт‑скиллам;
- правила проведения интервью;
- этические и профессиональные ограничения.

### 3. Карточки вакансий
Папка: `job_profiles/`

В проекте подготовлены три карточки:
- `sales_soft_client_001.md`
- `info_security_specialist_001.md`
- `design_engineer_rkd_001.md`

Каждая карточка содержит:
- основную информацию по вакансии;
- контекст роли;
- обязанности;
- обязательные и желательные требования;
- red flags;
- банк вопросов;
- рубрику оценки;
- порог прохождения;
- финальные вопросы кандидату.

### 4. Демо‑сценарии
Файл: `demo_scenarios.md`

Содержит короткие примеры интервью по всем трём вакансиям, которые можно использовать для демонстрации проекта.

### 5. Шаблон итогового отчёта
Файл: `candidate_evaluation_template.md`

Содержит универсальную форму, по которой ассистент может оформлять результат интервью по любой вакансии.

### 6. README проекта
Файл: `README.md`

Содержит описание структуры проекта, назначения файлов и порядка использования материалов.

## Почему проект сделан именно так

Проект собран не как один длинный документ, а как система отдельных модулей.

Это даёт несколько преимуществ:
- легче редактировать и обновлять конкретные вакансии;
- проще масштабировать проект;
- можно добавлять новые роли без изменений базовой логики;
- проще поддерживать единый стандарт проведения интервью;
- удобнее демонстрировать структуру проекта при сдаче.

## Как демонстрировать проект

Во время демонстрации можно показать следующую последовательность:

1. Открыть структуру проекта и показать ключевые файлы.
2. Показать системный промпт и объяснить, что именно он задаёт логику поведения ассистента.
3. Показать базу знаний компании.
4. Показать 1–2 карточки вакансий и объяснить, что каждая вакансия вынесена в отдельный профиль.
5. Показать демо‑сценарии интервью.
6. Показать шаблон итогового отчёта по кандидату.
7. Объяснить, что новые вакансии можно добавлять через новые `.md`‑карточки в папке `job_profiles`.

## Логика масштабирования

Если в будущем в проект нужно добавить новую вакансию, это делается так:
1. в папке `job_profiles` создаётся новый `.md`‑файл;
2. он оформляется по той же структуре, что и существующие карточки;
3. ассистент получает новый профиль без необходимости переписывать системный промпт целиком.

Таким образом, проект рассчитан не только на три текущие роли, но и на дальнейшее расширение.

## Практическая ценность проекта

Проект демонстрирует навыки:
- проектирования AI‑ассистента под прикладной бизнес‑сценарий;
- структурирования базы знаний;
- построения модульной архитектуры контента;
- подготовки сценариев интервью;
- разработки шаблона оценки кандидатов;
- продумывания масштабируемой структуры для дальнейшего роста решения.

## Итог

HR Interview Assistant — это модульный проект для AI‑поддержки интервью, в котором объединены:
- системный промпт;
- база знаний компании;
- карточки вакансий;
- демо‑сценарии;
- шаблон итоговой оценки кандидата.

Проект показывает, как можно превратить набор текстовых материалов в управляемую и расширяемую систему для проведения собеседований.
